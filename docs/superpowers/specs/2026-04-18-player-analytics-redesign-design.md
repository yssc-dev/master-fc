# 선수 분석 탭 재설계 Design

**Goal:** 선수 분석 탭의 지표 유의미성을 개선하고 중복·모호 탭을 정리한다. 우선순위는 **B (실력 평가/코칭) > C (팀 구성 가이드) > A (재미)**.

**Scope:** 8개 탭 모두 검토. 탭 수는 유지(8개), 그룹 헤더 추가, 일부 탭 내부 로직·라벨 개선.

---

## 배경

### 사용자 문제 제기

1. **시너지 TOP5가 전부 동률** (9경기 3승3무3패 50% × 5명) — 표본 부족 + tie-break 없음
2. **"케미"와 "시너지"가 의미적으로 중복**으로 보임
3. **현재 지표가 유의미한가** 전반적 의문

### 코드 실태 분석

| | 케미 | 시너지 |
|---|---|---|
| 소스 | 포인트로그 (득점 이벤트) | 앱 기록 경기 전체 |
| 수식 | 득점자+어시스트 쌍 **횟수** | 같은 팀일 때 **승률** |
| 임계값 | 없음 | `games >= 2` |
| 앱기록만? | 아니오 | 예 |

- **케미 vs 시너지**: 코드상 다르지만 UX상 구분 안 됨 (둘 다 "궁합" 뉘앙스)
- **골든콤비 vs 케미**: 동일 데이터(득점-어시스트 페어)의 다른 시각화. 단, 골든콤비는 득점 TOP5 중심, 케미는 전체 페어 랭킹이라 **저변이 다름**
- **시너지 단위 혼동**: 헤더 "3경기 기준"(세션 단위) vs 행 "9경기"(라운드 단위) — 같은 "경기" 단어가 다른 단위
- **시간대**: "첫 골 + 1시간" 기준은 근거 약함

## 아키텍처

### 탭 그룹화 + 순서 (신규)

```
[개인 분석]   선수카드 · 시즌레이스 · 키퍼킬러
[조합 분석]   시너지 · 골든콤비 · 득점콤비
[재미]       크로바/고구마 · 시간대
```

- 탭 칩(`<button>`) 가로배열 위에 작은 섹션 헤더(text-sm, gray-500) 3개
- 탭 수 변화 없음 (8개). "케미" → "득점콤비"로 탭 라벨 개명만 반영
- 탭 선택 상태(`activeTab`)는 기존 state 재사용, 그룹 헤더는 순수 표시용

### 데이터 플로우 (기존 유지)

- `events[]` = AppSync.getPointLog (포인트로그 시트)
- `playerLog[]` = AppSync.getPlayerLog (선수별집계기록로그 시트)
- `gameRecords[]` = AppSync.getHistory → parseGameHistory (앱 기록 stateJson blob)
- 세 소스의 탭별 사용은 현재와 동일

## 탭별 상세 변경

### [개인 분석] 선수카드 (4가지 추가)

**현재**: 공격(경기당 득점/도움), 수비(평균 실점), 승률

**추가 지표:**

#### 1. 추세선
- 최근 5경기 이동평균 vs 시즌 누적 평균 비교
- 차이에 따라 아이콘 표시:
  - 최근 평균 ≥ 시즌 평균 × 1.1 → 🔺 "상승세"
  - 최근 평균 ≤ 시즌 평균 × 0.9 → 🔻 "하락세"
  - 사이 → ➡️ "유지"
- 적용 지표: 득점, 도움 (수비는 샘플 적어 제외)
- 세션 수 < 5면 "추세 분석 불가" 표시

#### 2. GK / 필드 분리
- 현재 한 덩어리로 보여주는 지표를 "GK 출전 시" / "필드 출전 시" 2단으로 분리
- 분리 대상: 득점, 도움, 승률, 실점
- GK 출전 판정: `match.homeGk === playerName || match.awayGk === playerName`
- 각 카테고리 경기 수가 0이면 해당 섹션 "—"

#### 3. 상대적 위치
- 전체 팀원 평균 대비 백분율
- 표시 예시: `득점 +28% · 수비 -15% · 승률 +8%`
- 계산: `((나의 값 / 팀 평균) - 1) × 100`
- 음수면 "-", 양수면 "+" prefix. 소수점 반올림

#### 4. 출석률
- `참석 세션 수 / 전체 세션 수` (앱 기록 기준, 참석명단 시트 아님)
- 예: `출석 12/15 (80%)`
- 참석 판정: 해당 세션 gameRecords의 `teams[][]` flatten에 이름 존재하면 참석

### [개인 분석] 시즌레이스, 키퍼킬러 — 변경 없음

---

### [조합 분석] 시너지 (2가지 수정)

#### 1. 단위 라벨 명확화
- 상단 헤더 문구 (숫자는 예시):
  - Before: `앱 기록 3경기 기준 (날짜~날짜)`
  - After: `앱 기록 3세션 / 총 32라운드 기준 (날짜~날짜)` — 3은 실제 gameRecords.length, 32는 모든 세션의 main matches 총합
- 각 행 문구:
  - Before: `9경기 3승 3무 3패`
  - After: `9라운드 중 3승 3무 3패`

#### 2. tie-break 정렬
- 현재: 승률 기준 내림차순만 (동률 시 입력 순서 불안정)
- 변경: 동률 시 → 라운드 수 많은 순 → 이름 가나다 순
- Best TOP5: 승률 desc → 라운드수 desc → 이름 asc
- Worst TOP5: 승률 asc → 라운드수 desc → 이름 asc

### [조합 분석] 골든콤비 — 변경 없음

### [조합 분석] 득점콤비 (이전 "케미")

#### 1. 탭 라벨 개명
- 탭 칩 텍스트: "케미" → "득점콤비"
- 탭 내부 키도 함께 변경 (현재 키 값 확인 후 "combo" 또는 유사한 영문 식별자로 교체). 외부 저장 상태(localStorage 등)에 탭 키가 직렬화되는지 여부 확인하여 마이그레이션 필요 시 처리

#### 2. 경기 수 병기 + 효율% 정렬
- 기존 표시: `조재상+서라현 5회`
- 신규 표시: `조재상+서라현  5회 / 12경기 (42%)`
- "같이 뛴 경기 수" = gameRecords에서 두 선수가 같은 팀이었던 라운드 수 (시너지 계산과 동일 로직 재사용)
- 정렬 기준: **효율%** (= 조합 득점 횟수 / 같이 뛴 라운드 수) 내림차순
- tie-break: 효율 desc → 조합 횟수 desc → 이름 가나다 asc
- 같이 뛴 라운드가 < 3이면 제외 (표본 부족)

---

### [재미] 크로바/고구마

#### 1. 집계 로직 일반화
- 현재: `p.crova > 0` / `p.goguma < 0` 포인트 필드 기준 (playerLog)
- 변경: gameRecords 기반으로 "당일 1위 팀 / 꼴찌 팀" 소속 빈도 집계
- 이유: `useCrovaGoguma=false` 팀도 동일 통계 가능 + 포인트 필드 없이도 순위 계산 가능
- 순위 계산은 앱 내 기존 크로바/고구마 판정 로직을 재사용 (현재 `useCrovaGoguma=true` 팀에서 포인트 부여 시 사용하는 기준과 일치시킴). 없으면 승수 > 득실차 > 득점 순서로 랭크
- 동점 시 같은 순위 공동 기록

#### 2. UI 라벨 분기
- `getEffectiveSettings(team, "풋살").useCrovaGoguma`에 따라:
  - `true` → "Best 크로바 TOP5" / "Best 고구마 TOP5" (🍀/🍠 이모지 유지)
  - `false` → "승리팀 단골 TOP5" / "꼴찌팀 단골 TOP5" (이모지 없음 또는 🏆/📉)
- 탭 아이콘도 동일 분기

---

### [재미] 시간대 (재정의)

#### 기존 로직 제거
- "당일 첫 골 시각 + 1시간" 기준 초반/후반 분류 제거

#### 신규 로직: 라운드 미드포인트 기반

**계산 단계:**
1. 당일 `matches[]` 중 `isExtra === false`만 필터 → `mainMatches[]`
2. `N = mainMatches.length` (당일 총 메인 라운드 수)
3. 각 match의 배열 인덱스를 라운드 번호로 사용 (0-indexed)
4. 이벤트(골)의 `matchId`로 해당 라운드 인덱스 조회
5. `idx < Math.floor(N / 2)` → 초반, 그 외 → 후반

**표시:**
- 선수별 "초반 득점 / 후반 득점 / 초반 비율"
- 패턴 태그:
  - 초반 비율 ≥ 60% → "초반형 🔥"
  - 초반 비율 ≤ 40% → "후반형 ⚡"
  - 그 사이 → "균형형 ⚖️"
- 총 득점 < 5면 태그 대신 "샘플 부족" 표시

## 에러 / 엣지 케이스

### 표본 부족 처리
- 시너지: 라운드 < 2 → 미노출 (기존 동일)
- 득점콤비: 같이 뛴 라운드 < 3 → 미노출
- 추세선: 세션 < 5 → "분석 불가"
- 시간대 태그: 총 득점 < 5 → 태그 숨김

### 데이터 결측
- `gameRecords[]`가 비어있는 경우 → "앱 기록 경기가 없습니다" 배너 (앱기록 의존 탭만)
- `playerLog[]`가 비어있는 경우 → 기존 "데이터 없음" 유지

### useCrovaGoguma 플래그 누락
- 값이 `undefined`면 `false` 취급 (안전한 기본값)

## 테스트 전략

### 단위 테스트 (Vitest + jsdom)

**선수카드 추가 지표 (`PlayerCardTab` 유틸)**
- 이동평균 계산: 5세션 미만 → `null` 반환
- 상대적 위치: 팀 1명일 때 0% 반환 (divide-by-self)
- 출석률: 전체 0세션 시 `0/0` → `0%` 표시

**시너지 tie-break**
- 승률 동률 2쌍 → 라운드 수 많은 쪽이 상위
- 승률·라운드 모두 동률 → 이름 가나다 순

**득점콤비 효율%**
- `(goals / shared_rounds) * 100`, 공유 라운드 < 3 시 제외

**크로바/고구마 로직 일반화**
- 3팀 세션에서 동점 1위 → 공동 1위 처리
- `useCrovaGoguma=false` 팀에서도 정상 계산

**시간대 라운드 분류**
- `mainMatches.length = 10`일 때 idx 0~4 초반 / 5~9 후반
- `mainMatches.length = 9`일 때 idx 0~3 초반 / 4~8 후반
- `isExtra=true` 라운드 제외 확인

### 통합 테스트
- 탭 그룹 렌더 확인 (3개 헤더 + 8개 칩)
- 탭 클릭 시 해당 컴포넌트 마운트 확인
- `useCrovaGoguma` 플래그 전환 시 크로바/고구마 라벨 변화 확인
