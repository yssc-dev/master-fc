# 분석탭 V3 — 선수 중심 통합 + 신규 6지표 설계

- **작성일**: 2026-04-29
- **상태**: 설계 승인 대기
- **배경**: V2(2026-04-24)에서 트렌드·스트릭·PR·월랭킹·시너지매트릭스·골든트리오·어워드·🍀🍠를 도입했다. 사용자 사용 결과 두 가지 개선 요구가 누적됨:
  1. 개인 정보가 "선수카드"와 "명예의전당" 두 탭에 흩어져 매번 선수 선택을 두 번 해야 함 → 통합 필요
  2. 시너지·케미가 본인 기준으로 연계되지 않음 → 본인 카드 안에서 "나의 짝꿍" 형태로 노출 필요
  
  동시에 풋살 도메인 제약(매주 팀원 로테이션, 라운드별 5인 출전 미입력, GK만 라운드별 입력)을 반영해 산출 가능한 신규 지표 6종을 추가한다.

---

## 1. 도메인 제약 (재확인)

- **풋살 팀 정체성은 세션 단위**. 팀-팀 누적 승률·상대전적은 무의미. 모든 신규 지표는 **개인 능력** 또는 **선수간 조합** 축으로만 설계.
- **라운드별 5인 출전 명단 없음**. `로그_매치.our_members_json`은 그날 팀 로스터(보통 6~8명) 전체이며 라운드별 5인이 아님. 라운드 단위로 신뢰 가능한 식별자는 골/어시 이벤트의 `player`/`related_player`와 라운드별 GK(`our_gk`).
- 따라서 "5인 라인업 조합 승률", "본인 출전 라운드 vs 미출전 라운드 캐리지수" 같은 지표는 **산출 불가** → 후보에서 제외.

## 2. 목표

- 분석 탭을 **선수 중심**으로 재배치: 한 번의 선수 선택으로 본인 관련 정보가 모두 표시.
- 산출 가능한 신규 지표 6종(P3, P4, C2, C3, C4, C5)을 추가.
- 신규 지표 중 P3/P4는 어워드 탭에 전체 랭킹 카드로도 노출, 본인 카드에는 본인 랭킹 위치 캡션 표시.
- "조합" 탭(시너지매트릭스/케미/어워드/🍀🍠)은 현황표 성격 유지.

## 3. 비목표

## 6. 신규 지표 정의

### 6.1 P3 — 라운드 분포 (개인 + 어워드 전체)

**개인 view**: 한 세션 안에서 라운드 1·2·…·N에서의 본인 G/A 합계 막대그래프 + 회귀선.

**전체 슬로프 산출**:
- 입력: 해당 선수의 모든 세션 × 모든 라운드의 (round_idx, goals+assists) 점들
- 회귀선 기울기 `slope = Σ(x-x̄)(y-ȳ) / Σ(x-x̄)²`
- slope > 0 → 후반 가속형, slope < 0 → 초반 강자

**어워드 노출**:
- 「🏃 후반 폭격기」: slope 양수 TOP3
- 「🎯 초반 강자」: slope 음수 절댓값 TOP3
- 노출 임계: 누적 골+어시 ≥ 10 (표본 너무 작은 선수 제외)

**개인 카드 캡션**: 위 어워드 산출 결과를 재사용해 "후반 폭격기 N명 중 X위" 또는 "초반 강자 N명 중 X위". 둘 다 임계 미달 시 캡션 미표시.

**제약 보정**: 풋살은 본인이 그날 안 뛴 라운드를 알 수 없음. 그래서 **본인이 골/어시를 1회 이상 기록한 라운드만** 점으로 사용 (= 본인이 그 라운드에 뛴 게 거의 확실). G/A 0인 라운드는 표본에서 제외. 결과는 "활동한 라운드 중 후반에 더 활발한가"로 해석.

**중요 — 어워드 산출도 동일 규칙**: 위 슬로프 계산식은 어워드(후반폭격기/초반강자) 랭킹과 본인 view 회귀선 모두에 동일하게 적용한다. 즉 둘 다 G+A≥1 라운드만 점으로 쓴다.

### 6.2 P4 — 단독골 vs 받아먹은 골 (개인 + 어워드 전체)

**개인 view**: 도넛 차트
- 단독골 = `event_type=goal` 중 `related_player`가 비어있는 골
- 받아먹은 골 = `event_type=goal` 중 `related_player`가 채워진 골
- 절대값 + 비율 동시 표시

**어워드 노출**:
- 「🎯 혼자 박는 자」: 단독골 비율 TOP3
- 임계: 누적 골 ≥ 10 (표본 작은 선수 제외)

**개인 카드 캡션**: "혼자 박는 자 N명 중 X위".

### 6.3 C2 — 역시너지 페어 (시너지매트릭스 강조)

**산출**: V2 시너지매트릭스(N×N, 같은 팀 같은 라운드 승률)는 이미 모든 페어를 담고 있고 색도 초록/빨강/회색 3색 (`SynergyMatrixTab.jsx:31`). C2는 **역시너지 페어를 빠르게 찾는 진입점**을 추가하는 변경:
- 정렬 토글 추가: 기본(이름순) / 역시너지 TOP(같은 페어 승률 낮은 순) / 시너지 TOP(승률 높은 순)
- 정렬 모드일 때 매트릭스 셀이 "그 행에 대해 가장 낮은/높은 승률 페어"가 좌상단으로 오도록 행/열 동시 재배치
- 임계: 같은 팀 라운드 ≥ 5 (V2 그대로)

색상 변경은 없음 (V2 그대로). 정렬·재배치만 추가.

### 6.4 C3 — 어시 페어 TOP (케미 탭 sub-tab)

**산출**: 로그_이벤트에서 `event_type=goal AND related_player` 비어있지 않은 행을 (related_player, player) 페어 단위로 카운트.

**view**: TOP10 리스트 — 「{어시제공자} → {득점자}: N회」.

**임계**: 페어당 누적 ≥ 3회.

### 6.5 C4 — GK–필드 케미 (케미 탭 sub-tab)

**산출**: 로그_매치에서 (our_gk, 같은 라운드 our_members_json 내 필드 멤버) 페어 단위로 집계:
- 동행 라운드 수 = 그 페어가 한 행(=한 라운드)에 같이 등장한 횟수
- 무실점 라운드 수 = 위 라운드 중 `opponent_score === 0` (우리팀 그 라운드 실점 0) 인 라운드 수
- 무실점률 = 무실점 라운드 수 / 동행 라운드 수

`opponent_score`는 로그_매치 한 행(=한 라운드) 우리팀 실점값. 이 값이 0이면 그 라운드 GK 무실점.

**view**: GK 드롭다운 → 그 GK일 때 같이 뛴 필드 멤버별 무실점률 TOP/WORST 표.

**임계**: GK-필드 페어당 같은 라운드 ≥ 5.

**제약 보정**: 라운드별 필드 5인 출전이 없으므로 "같은 팀 로스터 안 멤버"로 근사. 즉 GK가 X일 때 같은 팀에 들어 있는 모든 멤버를 그 라운드 필드 동료로 간주. 결과 해석은 "X가 GK인 날 같이 한 팀이었던 동료가 누구일 때 무실점률이 높은가"이며 라운드 단위 정확도는 떨어짐을 캡션으로 명시.

### 6.6 C5 — 나의 짝꿍 (개인분석 탭 내부)

**산출**: V2 시너지매트릭스의 본인 row만 발췌. 같은 팀 같은 라운드 ≥ 5 페어만 표본.

**view**:
- 베스트 3 페어 (승률 높은 순)
- 워스트 3 페어 (승률 낮은 순)
- 각 페어에 라운드 수·승률 동시 표기

별도 매트릭스 컴포넌트 재사용. 새 컴포넌트는 "본인 row 추출 + 정렬"만.

## 7. 「케미」 탭 sub-tab 구조

기존 골든트리오 단일 view에 두 신규 카드를 더하면 너무 길어지므로 sub-tab으로 분리:

```
[탭 헤더] 골든트리오 | 어시페어 | GK케미
```

각 sub-tab은 독립 컴포넌트, 동일 부모로부터 `matchLogs` / `eventLogs` props 받음.

## 8. 「어워드」 탭 신규 카드

V2 카드 (불꽃/수호신/자책) 아래에 신규 두 카드 추가:

```
[기존]
  🔥 불꽃 (단일 세션 ≥ 3골) TOP5
  🛡 수호신 (세션 GK 무실점 달성) TOP5
  🤦 자책 누적 TOP3

[신규]
  🏃 후반 폭격기 / 🎯 초반 강자  TOP3 / TOP3 (양방향)
  🎯 혼자 박는 자                  TOP3
```

레이아웃은 기존 카드 grid 패턴 그대로.

- C4 GK-필드 케미는 라운드별 필드 5인 미입력 한계로 정확도가 낮음. 추후 풋살에 라운드별 출전 입력 기능이 추가되면 재계산 가능. 본 spec에서는 **결과 카드에 "그날 같은 팀 로스터 기준 근사"** 캡션을 명시.
- 축구 모드용 분석(formation/defenders) 미포함. 별도 spec.
