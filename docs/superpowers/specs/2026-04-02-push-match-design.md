# 밀어내기 대전 (Push Match) 설계 문서

## 개요

기존 2가지 경기 방식(대진표 대전, 자유대진 대전)에 **밀어내기 대전**을 세 번째 옵션으로 추가한다.

밀어내기는 1구장에서 3~6팀이 참가하며, 승리팀이 잔류하고 패배팀이 교체되는 방식이다. 관리자가 원하는 시점에 게임을 마감하면 기존과 동일한 순위/포인트 시스템이 적용된다.

---

## 1. 경기 모드 선택

### 변경 사항
- `matchMode`에 `"push"` 옵션 추가: `"schedule"` | `"free"` | `"push"`
- 밀어내기 선택 시 구장 수는 자동으로 1구장 고정 (선택 UI 비활성화)
- 팀 수는 3~6팀 선택 가능 (기존과 동일)

---

## 2. 밀어내기 전용 상태 데이터

`pushState` 객체를 게임 상태에 추가:

```
pushState: {
  winStreak: { teamIdx: number, count: number } | null,
  teamPlayCounts: { [teamIdx]: number },
  teamTotalGoals: { [teamIdx]: number },
  lastLoser: number | null,
  forcedRest: number | null,
  suggestedMatch: { home: number, away: number } | null,
}
```

### 필드 설명

| 필드 | 설명 | 갱신 시점 |
|------|------|-----------|
| `winStreak` | 현재 연승 중인 팀 인덱스와 연승 수 | 경기 확정 시 |
| `teamPlayCounts` | 팀별 누적 출전횟수 | 경기 확정 시 양 팀 +1 |
| `teamTotalGoals` | 팀별 누적 총 득점 (대진 우선순위용) | 경기 확정 시 |
| `lastLoser` | 방금 진 팀 인덱스 (다음 대진 제외) | 경기 확정 시 |
| `forcedRest` | 3연승으로 강제 휴식 중인 팀 인덱스 | 3연승 시 설정, 다음 경기 시작 시 해제 |
| `suggestedMatch` | 시스템 자동 제안 대진 (관리자 변경 가능) | 경기 확정 후 자동 계산 |

## 3. 다음 대진 자동 제안 알고리즘

### 첫 경기 (특수 케이스)
- 출전 기록 없음 → 팀 인덱스 0번 vs 1번 (시트 연동 순서)

### 경기 결과에 따른 분기

**Case A: 승리팀 득점 ≥ 2**
- 승리팀 잔류 (연승 카운트 +1)
- 패배팀 → `lastLoser`로 설정
- 연승 카운트가 3이 되면 → 승리팀도 `forcedRest`로 설정, 연승 리셋

**Case B: 승리팀 득점 < 2 (예: 1:0)**
- 두 팀 다 휴식
- 연승 리셋

**Case C: 무승부**
- 두 팀 다 휴식
- 연승 리셋

### 후보 풀 결정
- 전체 팀에서 제외: `lastLoser` + `forcedRest`
- 승리팀 잔류 시: 상대 1팀만 후보에서 선정
- 두 팀 다 휴식 시: 2팀을 후보에서 선정

### 상대팀 선정 우선순위
1. `teamPlayCounts` 적은 팀 우선
2. 동률이면 `teamTotalGoals` 많은 팀 우선
3. 득점도 같으면 팀 이름순 (가나다순)

### 3연승 강제 휴식 후
- 3연승팀(`forcedRest`)과 패배팀(`lastLoser`) 모두 제외
- 남은 팀 중 위 우선순위로 2팀 선정
- 다음 경기 시작 시 `forcedRest`는 null로 리셋 (후보 복귀)

### 엣지 케이스: 후보 부족 (3팀 + 3연승 등)
- 제외 후 후보가 2팀 미만이면 `lastLoser` 제한을 먼저 해제
- 그래도 부족하면 `forcedRest` 제한도 해제
- 이렇게 하면 최소 3팀 이상에서 항상 대진 구성 가능

## 4. PushMatchView UI 구성

### 파일
- `/src/components/game/PushMatchView.jsx` (신규)

### 상단 — 팀별 출전횟수 대시보드
- 모든 팀을 가로로 나열
- 각 팀 표시 항목: 팀명 / 출전횟수 / 총 득점
- 현재 경기 중인 2팀은 하이라이트
- 강제 휴식 팀은 "휴식" 뱃지 표시

### 중단 — 대진 영역
- 시스템 제안 대진 표시: `홈팀 vs 어웨이팀`
- 각 팀을 드롭다운 또는 버튼으로 변경 가능 (관리자 수동 대진 변경)
- 연승 중이면 "X팀 N연승 중" 표시

### 하단 — 경기 기록 영역
- 기존 `CourtRecorder` 컴포넌트 재사용
- GK 선택, 골/어시스트/자책골 기록
- "경기 확정" 버튼 → 결과 반영 → 다음 대진 자동 제안

### 마감
- 화면 하단에 "게임 마감" 버튼
- 누르면 summary 페이즈로 전환

## 5. 경기 흐름

### matchId 형식
- `"P{n}_C0"` (P = Push, n = 경기 번호, C0 = 1구장 고정)
- 포인트로그 표시: "1경기", "2경기" 등

### 라운드 진행
1. 대진 제안 표시 (첫 경기: 0번 vs 1번)
2. 관리자가 대진 확인 또는 변경
3. GK 선택 → 경기 기록 (CourtRecorder)
4. "경기 확정" 클릭
5. `completedMatches`에 결과 추가
6. `pushState` 갱신 (출전횟수, 연승, 득점 등)
7. 다음 대진 자동 계산 → 2번으로 돌아감
8. 관리자가 "게임 마감" → summary 페이즈

### reducer 액션 추가
- `SET_PUSH_MATCH`: 대진 설정/변경
- `CONFIRM_PUSH_ROUND`: 경기 확정 → pushState 갱신 → 다음 대진 계산

## 6. 기존 코드 재사용

| 기존 코드 | 용도 | 변경 필요 |
|-----------|------|-----------|
| `CourtRecorder` | 경기 기록 UI | 변경 없음 |
| `calcMatchScore()` | 스코어 계산 | 변경 없음 |
| `getTeamStandings()` | 순위 계산 (승3/무1/패0) | 변경 없음 |
| `calcPlayerPoints()` | 크로바/고구마 포인트 | 변경 없음 |
| `handleFinalize()` | 시트 기록 | matchId 포맷팅 분기 추가 ("P1_C0" → "1경기") |
| `useGameReducer` | 상태 관리 | pushState 관련 액션 추가 |
| `App.jsx` setup UI | 경기 모드 선택 | `"push"` 옵션 추가, 구장 수 1 고정 |

## 7. 순위 및 포인트

기존과 완전히 동일:
- 순위: 승(3점) / 무(1점) / 패(0점), 동률 시 골득실
- 팀 순위 점수: `teamCount - rankPosition` (4팀 기준 1등=4점, 4등=1점)
- 크로바: 1등 팀 선수에게 부여
- 고구마: 꼴찌 팀 선수에게 부여
- 시즌 리더 배율 규칙 동일 적용
