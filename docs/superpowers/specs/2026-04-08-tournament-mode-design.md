# 축구 대회 모드 설계

## 개요

기존 축구 리그(정기 경기) 외에 대회(토너먼트) 경기를 독립적으로 기록/관리하는 기능을 추가한다. 대회는 며칠에 걸친 일시적 이벤트로, 리그와 완전 분리된 데이터 공간을 가진다.

## 핵심 결정사항

| 항목 | 결정 |
|------|------|
| 대회 성격 | 며칠짜리 이벤트, 대진표가 미리 결정됨 |
| 대진 형태 | 대회마다 다름 — 풀리그 / 조별리그+녹아웃 / 녹아웃 / 자유(수동 대진) |
| 선수 구성 | 리그 선수단 + 대회별 게스트/용병 가능 |
| 리그와 관계 | 완전 분리 (대회 기록은 리그 통계에 합산 안 됨) |
| 타팀 경기 | 스코어만 수동 입력 (우리팀 경기만 상세 기록) |
| 공유 | 1차: 우리팀 전용, 추후 다른 팀 공유 확장 가능 |

## UI 흐름

### 진입 경로

```
팀 대시보드 (하버FC)
  ├── 대시보드 탭 (리그)
  ├── 개인기록 탭 (리그)
  ├── 분석 탭 (리그)
  ├── 경기관리 탭 (리그)
  └── 대회 탭 ← 신규
        ├── 대회 목록 (진행중 / 완료)
        ├── [+ 새 대회] → 대회 생성
        └── [대회 선택] → 대회 대시보드 (4탭)
              ├── 대시보드
              ├── 개인기록
              ├── 분석
              └── 경기관리
```

### 대회 생성 (관리자)

1. 대회명 입력 (예: "2026 여름 챔피언스컵")
2. 기간 입력 (시작일 ~ 종료일)
3. 참가팀 입력 (우리팀 자동 포함 + 상대팀 이름 추가)
4. 대진 형태 선택: 풀리그 / 조별리그+녹아웃 / 녹아웃 / 자유(수동)
5. 대진표 생성 (형태에 따라 자동 생성 또는 수동 입력)
6. 생성 → 구글시트에 대회 시트 탭 자동 생성 + Firebase 메타 저장

### 대회 대시보드

**대시보드 탭:**
- **팀 순위표**: 전체 참가팀 순위 (경기수, 승, 무, 패, 득점, 실점, 승점)
  - 우리팀 경기: 앱에서 기록한 결과로 자동 반영
  - 타팀 경기: 수동 입력한 스코어로 반영
- **경기 일정**: 전체 대진표 날짜순 표시
  - 우리팀 경기 하이라이트 (배경색 구분)
  - 완료 경기는 스코어 표시, 미완료 경기는 "예정" 표시
- **개인 TOP**: 득점왕, 어시왕, 클린시트 상위 3명 간단 표시

**개인기록 탭:**
- 우리팀 선수 전원의 대회 기록
- 컬럼: 선수명, 경기수, 골, 어시, 클린시트, 포인트
- 포인트 = 골+1, 어시+1, 자책-1, 클린시트+1 (리그와 동일)

**분석 탭:**
- 골든콤비, 시즌레이스, 케미 — 대회 이벤트로그 기반
- 리그 분석과 동일 구조, 대회 데이터만 사용

**경기관리 탭:**
- **우리팀 경기**: 기존 축구 모드와 동일 흐름
  - 일정에서 경기 선택 → 참가 선수 선택 (리그 명단 + 용병 추가 가능)
  - 라인업 11명 + GK/DF 지정
  - 이벤트 기록 (골/상대골/교체) → 경기 종료
- **타팀 경기**: 스코어만 입력
  - 일정에서 타팀 경기 선택 → 홈스코어, 원정스코어 입력 → 저장
- 결과가 팀 순위표에 자동 반영

## 데이터 저장

### 구글시트 (기존 시트에 대회별 탭 추가)

| 시트명 | 용도 | 비고 |
|-------|------|------|
| `대회_목록` | 대회 메타정보 | 1개 시트, 모든 대회 목록 |
| `대회_{ID}_일정` | 전체 대진표 + 스코어 | 대회별 1개 |
| `대회_{ID}_이벤트로그` | 우리팀 경기 로우데이터 | 대회별 1개 |
| `대회_{ID}_선수기록` | 우리팀 선수별 대회 집계 | 대회별 1개 |

### 대회_목록 컬럼

| 컬럼 | 설명 |
|------|------|
| 대회ID | 고유 ID (예: "t_1712345678") |
| 대회명 | 대회 이름 |
| 시작일 | 시작 날짜 |
| 종료일 | 종료 날짜 |
| 참가팀 | 쉼표 구분 팀 목록 (우리팀 포함) |
| 대진형태 | fullLeague / groupKnockout / knockout / manual |
| 상태 | active / finished |
| 생성시간 | 생성 timestamp |

### 대회_{ID}_일정 컬럼

| 컬럼 | 설명 |
|------|------|
| 경기번호 | 순번 |
| 날짜 | 경기 날짜 |
| 라운드 | 라운드명 (조별1R, 준결승 등) |
| 홈팀 | 홈팀 이름 |
| 원정팀 | 원정팀 이름 |
| 홈스코어 | 홈팀 득점 (미입력이면 빈칸) |
| 원정스코어 | 원정팀 득점 |
| 우리팀여부 | Y/N (우리팀 경기인지) |
| 상태 | scheduled / finished |

### 대회_{ID}_이벤트로그 컬럼 (로우데이터)

| 컬럼 | 설명 |
|------|------|
| 경기번호 | 일정의 경기번호 참조 |
| 상대팀명 | 상대팀 이름 |
| 이벤트 | 출전 / 골 / 자책골 / 실점 / 교체 |
| 선수 | 주체 선수 (출전자, 득점자, 실점GK, 교체IN) |
| 관련선수 | 어시스트, 교체OUT 선수 |
| 포지션 | GK / DF / FW (출전, 교체 시 승계포지션) |
| 입력시간 | 이벤트 발생 timestamp |

### 대회_{ID}_선수기록 컬럼

| 컬럼 | 설명 |
|------|------|
| 선수명 | 선수 이름 |
| 전체경기 | 출전 경기 수 |
| 필드경기 | 필드 출전 수 |
| 키퍼경기 | GK 출전 수 |
| 골 | 득점 수 |
| 어시 | 어시스트 수 |
| 클린시트 | 클린시트 횟수 |
| 실점 | 실점 수 (GK일 때) |
| 자책골 | 자책골 수 |
| 포인트 | 총 포인트 |

### Firebase

```
tournaments/{팀이름}/{대회ID}: {
  name, startDate, endDate, teams: [], format,
  status: "active" | "finished",
  createdAt, createdBy,
}

// 진행중 경기 상태 (우리팀 경기 기록 중)
tournaments/{팀이름}/{대회ID}/activeGame: {
  matchNum, state: JSON, updatedAt
}
```

### Apps Script

- `createTournament` — 대회 생성 + 시트 탭 자동 생성
- `getTournamentList` — 대회 목록 조회
- `getTournamentSchedule` — 대진표/일정 조회
- `updateMatchScore` — 타팀 경기 스코어 입력
- `writeTournamentEventLog` — 우리팀 경기 이벤트로그 쓰기
- `writeTournamentPlayerRecord` — 우리팀 선수기록 쓰기
- `getTournamentPlayerRecords` — 선수기록 조회
- `getTournamentEventLog` — 이벤트로그 조회
