# 경기 중 팀 명단 수정 기능

## 개요

대진표 모드 경기 진행 중, 팀원 불참·신규 참여가 생긴 경우 해당 선수들만 팀에 추가/제거할 수 있는 편집 기능. 스네이크 드래프트 결과와 과거 경기 기록은 보존한다.

## 진입점

경기 화면의 "팀명단" 모달 **상단**에 `팀 수정` 버튼 추가.

클릭 시 동작:

- `phase`를 `"teamBuild"`로 전환 (기존 팀 편성 화면 재활용)
- `schedule`, `completedMatches`, `allEvents`, `confirmedRounds`, `currentRoundIdx`, `viewingRoundIdx` 그대로 유지
- 편집 진입 시점에 `teams`, `attendees`, `gks` 스냅샷 저장 (취소 시 복원용)
- `draftMode`는 `"free"`로 강제 설정 (자유 편성 UX 재활용)
- 편집 모드 플래그 (예: `teamEditMode: true`) 세팅

## 편집 화면 동작

기존 자유 편성 화면을 재활용하되, **편집 모드 플래그가 true일 때** 제약을 추가한다.

**허용 동작:**
- 팀별 선수 ✕ 버튼으로 제거 → 미배정 풀로 이동
- 미배정 풀 클릭 → 선택된 팀에 추가

**미배정 풀 구성:**
- 기존 `attendees` 중 어느 팀에도 속하지 않은 선수
- **시즌 선수 중 `attendees`가 아닌 선수** (불참 선수 목록)
- **`+ 새 선수 입력`** 필드로 시즌 외 이름 직접 추가 (0p 게스트)
  - 추가 시 `attendees`에 포함되고 선택된 팀에 배정

**차단 (편집 모드에서만):**
- 팀 개수 변경 (스케줄 무효화 방지)
- 팀 이름 변경
- 팀 색상 변경
- 스네이크 드래프트 "재배치" 버튼

**편집 모드 UI:**
- 상단에 "경기 진행 중 · 편집 모드" 뱃지
- `완료` 버튼 + `취소` 버튼 (기존 "경기 시작" 버튼 자리를 조건부로 교체)

## 완료

- 편집 중인 `teams`, `attendees` 그대로 확정
- `gks[i]`의 선수가 `teams[i]`에서 빠졌으면 해당 키 삭제
- `teamEditMode` 해제, 편집 스냅샷 폐기
- `phase` → `"match"` 복귀
- 현재 라운드/스케줄/경기기록 유지
- **Google Sheets 동기화 스킵** — 집계 완료 시 최종 명단만 반영

## 취소

- 편집 진입 시 저장한 스냅샷(`teams`, `attendees`, `gks`)으로 복원
- `teamEditMode` 해제, 편집 스냅샷 폐기
- `phase` → `"match"` 복귀
- 편집 세션 내 모든 변경 무효 (단, 완료되어 이미 반영된 과거 편집은 건드리지 않음)

## 기록 영향

- `completedMatches`, `allEvents` 무변경
- 제거된 선수의 과거 득점·도움·경기수는 통계에 그대로 반영 (실제로 뛴 기록이므로 정상)
- 추가된 선수는 수정 시점 이후 라운드부터 참여

## 유지 (변경 없음)

- `snakeDraft` 로직
- `schedule` 배열 (팀 인덱스 기반이라 선수 변경과 무관)
- `CONFIRM_ROUND` / `UNCONFIRM_ROUND` 리듀서
- 기존 `phase === "teamBuild"` 자유 편성 UX (일반 편성 플로우)

## 영향 범위

- **파일**: `src/App.jsx`, `src/hooks/useGameReducer.js`
- **신규 state**: `teamEditMode` (boolean), `teamEditSnapshot` (객체)
- **신규 액션**: `ENTER_TEAM_EDIT`, `EXIT_TEAM_EDIT_SAVE`, `EXIT_TEAM_EDIT_CANCEL`
- **UI 변경**:
  - 팀명단 모달 상단에 `팀 수정` 버튼 추가
  - 팀 편성 화면 상단에 편집 모드 뱃지 + 완료/취소 버튼 조건부 표시
  - 팀 편성 화면에서 편집 모드일 때 팀 이름/색/재배치 UI 비활성화
  - 미배정 풀에 불참 시즌 선수 포함 + 수기 입력 필드

## 동작 검증 (케이스별)

| 케이스 | 입력 | 결과 |
|--------|------|------|
| 제거 | R5 진행 중 A팀 선수 "김철수" 제거 → 완료 | R6부터 김철수 없음, R1~R5 득점 기록 그대로 |
| 추가 (시즌 불참자) | "이영희" 선택 → B팀 → 완료 | B팀에 이영희 추가, `attendees`에 포함 |
| 추가 (게스트) | "홍길동" 수기 입력 → C팀 → 완료 | C팀에 홍길동(0p) 추가, `attendees`에 포함 |
| GK 제거 | `gks[0] === "김철수"`, 김철수 제거 → 완료 | `gks[0]` 삭제 |
| 취소 | 두세 명 변경 후 취소 | 편집 진입 시점 상태로 완전 복원 |
| 과거 편집은 복원 안 됨 | 편집 A 완료 → 편집 B 시작 → B 취소 | A의 변경은 그대로, B만 무효 |

## 테스트 시나리오

1. 대진표 모드 R1 확정 → 팀수정 → A팀 임의 선수 제거 → 완료 → R2 대진에서 해당 선수 없음
2. 팀수정 → 시즌 불참자 목록에서 한 명 추가 → B팀 선택 후 클릭 → 완료 → B팀에 추가
3. 팀수정 → 수기 입력 "게스트A" → C팀 → 완료 → C팀에 0p 추가
4. 팀수정 → GK 지정된 선수 제거 → 완료 → 해당 팀 GK 해제 확인
5. 팀수정 → 여러 변경 후 취소 → 원상복귀
