# 빠른 기록 UI (Quick Record) 설계 문서

## 개요

기존 CourtRecorder를 대체하는 새 경기 기록 UI. 음성 인식과 간소화된 터치 입력으로 골/어시/자책골 기록 속도를 개선한다. 모든 경기 방식(대진표/자유대진/밀어내기)에서 사용.

### 핵심 문제
현장에서 "철수 골 영희 어시" 음성을 듣고 기록할 때, 기존 UI는 구장(A/B) → 팀(홈/어웨이) → 선수 순서로 풀스캔이 필요했음.

### 해결 방향
1. **음성 입력**: 🎤 Push-to-talk으로 "이름 골 이름 어시" 말하면 자동 기록
2. **터치 입력**: 선수 옆 GK/⚽ 버튼 → 1탭으로 어시/단독/자책 선택 완료

---

## 1. 입력 모드

### 음성 모드 (Web Speech API)
- 🎤 버튼을 **꾹 누르는 동안** 음성 인식 활성화 (Push-to-talk)
- 버튼은 화면 하단에 고정 (1개, 전 구장 공통)
- 음성 → 텍스트 → 파싱 → 선수 매칭 → 이벤트 기록

### 터치 모드
- 각 선수 옆에 [GK] [⚽] 버튼
- GK 버튼: 탭하면 해당 선수를 GK로 지정 (토글)
- ⚽ 버튼: 탭하면 인라인으로 어시 선택지 전개

---

## 2. 터치 모드 상세

### GK 지정
- 선수 이름 왼쪽에 [GK] 버튼
- 탭하면 해당 팀의 GK로 지정 (기존 GK는 자동 해제)
- GK 지정된 선수는 시각적으로 구분 (예: "GK" 뱃지)

### 골 기록 (⚽ 버튼)
- **선행 조건**: 양 팀 GK가 모두 지정되어야 함. 미지정 시 "GK를 먼저 지정하세요" 안내
- ⚽ 탭 후 인라인 전개:

```
[GK] 조재상  [⚽] ← 눌린 상태
  어시: [김성환] [오희종] [채수찬] [어시없음] [자책골]
```

- 같은 팀 선수 목록 + [어시없음] + [자책골] 이 한 줄에 표시
- 선수 탭 → 골 + 어시 기록
- [어시없음] → 단독골 기록
- [자책골] → 자책골 기록 (상대팀에 득점)
- 1탭으로 완료

---

## 3. 음성 인식 상세

### 기술
- Web Speech API (`SpeechRecognition`) 사용
- 한국어 (`ko-KR`) 설정
- Push-to-talk: 버튼 누르는 동안만 인식

### 텍스트 파싱
키워드 기반 규칙 파싱. 순서 무관.

| 음성 | 파싱 결과 |
|------|-----------|
| "조재상 골 채수찬 어시" | 골: 조재상, 어시: 채수찬 |
| "채수찬 어시 조재상 골" | 골: 조재상, 어시: 채수찬 |
| "조재상 골" | 골: 조재상, 어시 없음 |
| "조재상 자책" | 자책골: 조재상 |
| "조재상 자책골" | 자책골: 조재상 |

### 선수명 매칭
- 참가 선수 전체 목록(전 구장)에서 fuzzy matching
- 정확히 1명 매칭 → 자동 확정
- 2명 이상 후보 → 선택지 팝업 ("조재상? 조승훈?")
- 매칭 실패 → "인식 실패" 표시

### 구장 자동 판별
- 매칭된 선수가 속한 구장을 자동으로 판별 (선수는 동시에 2구장에 뛸 수 없음)
- 해당 구장의 matchId로 이벤트 기록

### 실점 GK 자동 지정
- 골 기록 시, 상대팀 GK가 지정되어 있으면 자동으로 실점 GK 설정

### 인식 결과 표시
- 성공 → 토스트 2초 표시 + 이벤트 로그에 추가
- 실패/모호 → 후보 선택 UI 또는 "인식 실패" 표시

## 4. 레이아웃

### 1구장 (밀어내기, 자유대진 1코트 등)
```
── 스코어보드 ─────────────────
  팀보영  0 : 0  팀재상

── 선수 목록 ──────────────────
  팀보영                 팀재상
  [GK] 김성환 [⚽]  |  [GK] 오희종 [⚽]
  [GK] 김종헌 [⚽]  |  [GK] 조재상 [⚽]
  ...                    ...

── 🎤 꾹 눌러서 말하기 ─────────
── 이벤트 로그 ────────────────
```

### 2구장 (대진표 4팀 2코트 등)
```
── A구장 ──────────────────────
  스코어보드 + 선수 목록 + GK/골 버튼

── B구장 ──────────────────────
  스코어보드 + 선수 목록 + GK/골 버튼

── 🎤 꾹 눌러서 말하기 ─────────  (1개, 전구장 공통)
── 이벤트 로그 ────────────────
  ⚽ 조재상(골) ← 채수찬(어시) [A구장]
  ⚽ 남인진(골) 단독 [B구장]
```

## 5. 기존 코드 영향

### 대체 대상
- `src/components/game/CourtRecorder.jsx` → 새 컴포넌트로 대체

### 재사용
- `calcMatchScore()` — 스코어 계산 변경 없음
- 이벤트 구조 (`allEvents`) — 기존과 동일한 형식으로 기록
- `onRecordEvent`, `onUndoEvent`, `onDeleteEvent`, `onEditEvent` — 기존 콜백 그대로 사용

### 호출부 수정
- `ScheduleMatchView.jsx` — CourtRecorder → 새 컴포넌트로 교체
- `FreeMatchView.jsx` — 동일
- `PushMatchView.jsx` — 동일
- `App.jsx` — import 변경

### 이벤트 데이터 구조 (변경 없음)
```js
{
  id: "evt_{ts}_{seq}",
  type: "goal" | "owngoal",
  matchId: "R1_C0" | "F1_C0" | "P1_C0",
  player: "조재상",        // 득점/자책 선수
  assist: "채수찬" | null, // 어시스트 선수
  team: "팀재상",          // 소속팀
  scoringTeam: "팀재상",
  concedingTeam: "팀보영",
  concedingGk: "김성환",
  courtId: 0,
  timestamp: number,
}
```

## 6. Web Speech API 호환성

| 브라우저 | 지원 |
|----------|------|
| Chrome (Android/Desktop) | 완전 지원 |
| Safari (iOS 14.5+) | 부분 지원 (webkitSpeechRecognition) |
| Firefox | 미지원 |

- 미지원 브라우저: 🎤 버튼 숨김, 터치 모드만 사용
- `window.SpeechRecognition || window.webkitSpeechRecognition` 체크
