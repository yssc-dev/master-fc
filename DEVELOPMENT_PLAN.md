# 구기종목 기록 플랫폼 — 개발 플랜

> 취미 구기종목(풋살, 축구, 야구, 농구 등) 팀의 경기 기록 플랫폼
> 최종 업데이트: 2026-03-13

---

## 현재 상태 (v0.1 — 마스터FC 전용)

- 단일 HTML + Google Apps Script + Google Sheets
- Firebase Realtime DB로 실시간 동기화
- 풋살 전용, 커스텀 포인트 시스템 (크로바/고구마 등)
- 인증: 관리자가 시트에 직접 등록 → 이름+번호로 로그인
- 멀티팀 구조 적용 완료 (팀별 데이터 분리)

---

## 소프트웨어 설계

### 데이터 모델

```
Users (가입 유저)
├── id
├── name
├── phone
├── auth_method          ← 현재: 관리자등록 / 추후: 이메일, 카카오, 구글
├── created_at
└── 1명이 여러 팀 소속 가능

Teams (팀)
├── id
├── name                 ← "마스터FC"
├── created_by (user_id) ← owner
├── invite_code          ← 6자리, 팀원 초대용
├── created_at
└── 1팀이 여러 종목 구독 가능

TeamMembers (팀-유저 소속)
├── user_id
├── team_id
├── role                 ← "owner" | "admin" | "member" | "viewer"
├── back_number
├── joined_at
└── 복합키: user_id + team_id

TeamSports (팀-종목 구독)
├── team_id
├── sport_type           ← "futsal" | "soccer" | "baseball" | "basketball"
├── scoring_rule_id      ← 집계방식 (공식룰 or 커스텀)
├── is_active            ← 과금/활성 상태
├── activated_at
└── 복합키: team_id + sport_type
   (마스터FC가 풋살+축구 둘 다 기록하고 싶으면 행 2개)

SportTypes (종목 정의 — 시스템 테이블)
├── id: "futsal" | "soccer" | "baseball" | "basketball"
├── label: "풋살" | "축구" | "야구" | "농구"
├── default_events       ← ["goal","assist","clean_sheet","own_goal",...]
├── default_scoring      ← { "goal": 3, "assist": 1, "own_goal": -2 }
├── team_size            ← { min: 4, max: 5, default: 5 }
├── round_label          ← "쿼터" | "이닝" | "세트" | "라운드"
└── 종목 추가 = 행 추가 + UI 컴포넌트 개발

ScoringRules (집계방식)
├── id
├── sport_type
├── team_id              ← null이면 공식룰 (시스템 기본)
├── name                 ← "공식룰" | "마스터FC 커스텀"
├── rules_json           ← { "goal": 3, "assist": 1, "crova": 5, "goguma": -3 }
├── custom_events        ← [{ key:"crova", label:"크로바", desc:"..." }, ...]
└── is_default
   공식룰: team_id = null, 종목별 1개
   커스텀: team_id 지정, owner/admin이 편집

Games (경기)
├── id
├── team_id
├── sport_type
├── game_date
├── status               ← "진행중" | "확정"
├── state_json           ← 전체 경기 상태 (라운드, 팀구성, 이벤트 등)
├── summary
├── created_by (user_id)
├── created_at
├── finalized_at
└── 같은 날 여러 경기 가능

GameEvents (이벤트 — 정규화된 개별 기록)
├── id
├── game_id
├── round_number
├── player_id (user_id)
├── event_type           ← "goal" | "assist" | "crova" | 커스텀 이벤트
├── point_value          ← 집계방식에 따라 계산된 점수
├── timestamp
└── 통계/리더보드 집계용

PlayerStats (시즌 집계 캐시 — 경기 확정 시 갱신)
├── user_id
├── team_id
├── sport_type
├── season               ← "2026" or "2026-S1"
├── games, goals, assists, points, ...
└── 빠른 조회용 (원본은 GameEvents)
```

### 관계 요약

```
User ──< TeamMember >── Team
                          │
                     TeamSports ──> SportType
                          │              │
                     ScoringRules ───────┘
                          │
                        Games
                          │
                     GameEvents ──> User (player)
                          │
                     PlayerStats
```

### 권한 체계

| 기능 | owner | admin | member | viewer |
|------|:-----:|:-----:|:------:|:------:|
| 팀 삭제 | O | | | |
| 팀 설정 변경 (이름, 종목 추가) | O | | | |
| 집계방식 변경 | O | O | | |
| 팀원 초대/추방 | O | O | | |
| 역할 변경 | O | O* | | |
| 경기 생성 | O | O | O | |
| 경기 기록 (실시간) | O | O | O | |
| 기록 확정/삭제 | O | O | | |
| 경기 조회 | O | O | O | O |
| 통계 조회 | O | O | O | O |

*admin은 member↔viewer만 변경 가능, admin 지정은 owner만

- **owner**: 팀당 1명 (생성자), 양도 가능
- **admin**: 여러 명 가능, 운영 권한
- **member**: 일반 팀원, 기록 참여 가능
- **viewer**: 읽기 전용 (가족, 서포터 등)

### 유저 플로우

#### 1. 가입 및 팀 진입

```
[가입/로그인]
  ├── 현재: 관리자가 회원인증 시트에 등록 → 이름+번호 로그인
  └── 추후: 이메일/카카오/구글 소셜 로그인
        ↓
[라우팅 분기]
  ├── 소속 팀 0개 → 홈 (팀 만들기 / 초대코드)
  ├── 소속 팀 1개 → 팀 대시보드로 바로 입장 (자동 스킵)
  └── 소속 팀 여러개 → 홈 (팀 선택)
        ↓
[홈 — 내 팀/종목 목록]
  ├── 팀/종목 카드 선택 → 팀 대시보드 입장
  ├── [팀 만들기] → 팀 생성 플로우
  ├── [초대코드 입력] → 기존 팀 합류
  └── ⚙️ 설정: "기본 팀 지정" → 다음 로그인부터 자동 스킵
```

**자동 스킵 설정**:
- 유저가 홈에서 특정 팀을 "기본 팀"으로 지정 가능
- 기본 팀 설정 시: 로그인 → 팀 대시보드 직행 (홈 건너뜀)
- 대시보드 상단에 [팀 전환] 버튼으로 언제든 홈 복귀 가능
- 소속 1팀이면 자동으로 기본 팀 처리 (설정 없이 스킵)
- 종목 선택은 대시보드 안의 종목 탭에서 전환

#### 2. 팀 만들기

```
[팀 만들기]
  ├── 팀이름 입력 (예: "마스터FC")
  ├── 기본 종목 선택 (풋살/축구/야구/농구 중 1개)
  │   └── 추가 종목은 팀 설정에서 나중에 추가 (과금 포인트)
  ├── 집계방식 선택
  │   ├── [공식룰 사용] → 종목별 기본 점수 체계 자동 적용
  │   └── [커스텀] → 이벤트별 점수 직접 설정
  │       ├── 기본 이벤트 (골, 어시, 클린시트 등) 점수 수정
  │       └── 커스텀 이벤트 추가 (크로바, 고구마 등)
  ├── 생성 완료 → 자동으로 owner + 초대코드 발급
  └── 초대코드 공유 (카카오톡, 링크 복사 등)
```

#### 3. 팀원 합류

```
[초대코드 입력] 또는 [초대 링크 클릭]
  ├── 미가입 → 가입 후 자동 합류
  └── 가입 유저 → 바로 합류 (기본 role: member)
        ↓
owner/admin이 팀원 관리에서:
  ├── 역할 변경 (member → admin 등)
  ├── 등번호 설정
  └── 추방
```

#### 4. 홈 — 내 팀 목록

```
[홈 화면]
  ┌─────────────────────────┐
  │  내 팀                   │
  │                         │
  │  ┌───────────────────┐  │
  │  │ 마스터FC            │  │  ← 팀 카드 (종목은 대시보드 안에서)
  │  │ 멤버 12명 · 풋살,축구│  │     활성 종목 미리보기
  │  └───────────────────┘  │
  │  ┌───────────────────┐  │
  │  │ 동네형들            │  │  ← 다른 팀
  │  │ 멤버 8명 · 농구     │  │
  │  └───────────────────┘  │
  │                         │
  │  [+ 팀 만들기]  [초대코드] │
  └─────────────────────────┘

팀 카드 탭 → 팀 대시보드 입장
팀 1개면 자동 스킵 (기본 팀 설정 가능)
```

#### 5. 팀 대시보드 (입장 후 메인 화면)

팀 = 사람 모임. 종목은 그 팀이 하는 활동.
팀원 정보는 종목 간 공유, 기록/통계/경기는 종목별 분리.

```
[마스터FC]
  ┌─────────────────────────────────┐
  │  마스터FC              [팀 전환] │
  │  ─────────────────────────       │
  │                                 │
  │  ── 종목 탭 ──                   │
  │  ┌──────────────────────────┐   │
  │  │ [⚽ 풋살] [⚽ 축구] [+ 추가]│   │  ← 종목 탭 전환
  │  └──────────────────────────┘   │
  │  (+ 추가 = owner만, 과금 포인트)  │
  │                                 │
  │  ── 공통 영역 (종목 무관) ──      │
  │                                 │
  │  📋 전체 팀원 (12명)             │
  │  ┌──────────────────────────┐   │
  │  │ #8  조재상  관리자         │   │
  │  │ #1  조승훈  멤버          │   │
  │  │ #44 채수찬  멤버          │   │
  │  │ ...                      │   │
  │  └──────────────────────────┘   │
  │  (팀원은 모든 종목에서 공유)      │
  │                                 │
  │  ── 종목별 영역 (선택된 탭) ──    │
  │                                 │
  │  📊 풋살 시즌 누적 기록 (2026)   │
  │  ┌──────────────────────────┐   │
  │  │ 경기수: 42  │ 총 골: 187  │   │
  │  │ 1위 조재상 42pt          │   │
  │  │ 2위 조승훈 38pt          │   │
  │  │ [전체 리더보드 보기]       │   │
  │  └──────────────────────────┘   │
  │                                 │
  │  ┌──────────────────────────┐   │
  │  │                          │   │
  │  │   🏟️  풋살 경기 시작      │   │  ← 메인 CTA (종목별)
  │  │                          │   │
  │  └──────────────────────────┘   │
  │                                 │
  │  📂 풋살 과거 경기 조회          │
  │                                 │
  │  ── 하단 공통 ──                 │
  │  ⚙️ 팀 설정 (owner/admin)       │
  │                                 │
  └─────────────────────────────────┘

종목 탭 전환 시:
  축구 탭 선택 → 시즌기록/경기시작/과거조회가 축구 데이터로 변경
  팀원 명단은 그대로 유지

진행중 경기 있으면:
  ├── 🟢 "이어하기" 버튼 표시 (경기 시작 위에)
  └── "경기 시작"은 관리자만 (기존 경기 덮어쓰기 경고)
  └── 진행중 표시는 해당 종목에만 (풋살 진행중이어도 축구는 새로 가능)
```

**종목 탭 설계 원칙**:
- 공유: 팀원 명단, 팀 설정, 초대코드
- 분리: 경기 기록, 시즌 통계, 과거 조회, 리더보드, 집계방식
- 종목 간 독립: 풋살 진행중이어도 축구 경기 시작 가능
- 종목 추가는 owner만 (추후 과금 포인트)

#### 6. 시즌 누적 기록 상세

```
[전체 리더보드 보기]
  ├── 시즌 선택 (2026, 2025, ...)
  ├── 정렬 기준 (포인트, 골, 어시, 출석률 등)
  │
  │  순위  선수     경기  골  어시  포인트
  │  1     조재상   10   15   8    42
  │  2     조승훈    8   12   6    38
  │  3     정보영   10    9   5    33
  │  ...
  │
  ├── 선수 이름 탭 → 개인 상세 기록
  │   ├── 시즌 요약 (경기수, 골, 어시, 포인트, 출석률)
  │   ├── 경기별 기록 리스트
  │   ├── 크로바/고구마 누적
  │   └── 트렌드 (최근 5경기 포인트 추이)
  │
  └── 종목별 특수 리더보드
      ├── 크로바 시즌 랭킹 (커스텀 이벤트)
      ├── 고구마 시즌 랭킹
      └── 클린시트 랭킹 (GK)
```

**설계 포인트**:
- 날짜별 그룹핑 + 같은 날 여러 경기는 시간으로 구분
- 경기 카드에 핵심 요약 (라운드수, 참석인원, 우승팀, MVP) 미리 표시
- 상세 진입 후 탭 3개로 관점 전환 (대진결과 / 개인기록 / 팀순위)
- 전부 읽기전용 (수정 불가 명시)
- 시즌/월 필터로 빠른 탐색

#### 8. 경기 기록 (풋살 예시 — 현재 구현된 플로우)

```
[경기 시작]
  ├── 참석자 선택 (명단에서 체크 or 수동 추가)
  ├── 팀 수 설정 → 스네이크 드래프트 팀 편성
  ├── 대진표 자동 생성
  ↓
[라운드 진행]
  ├── GK 선택
  ├── 실시간 이벤트 기록 (골, 어시, 자책골 등)
  ├── 라운드 확정 → 다음 라운드
  ├── 실시간 동기화 (여러 기기 동시 접속)
  ↓
[전체 라운드 완료]
  ├── 크로바/고구마 등 보너스 이벤트 집계
  ├── 최종 결과 확인 (개인기록, 팀순위)
  └── 기록 확정 → 영구 저장 → 시즌 누적에 반영
```

#### 설계 포인트

- **공식룰로 시작, 커스텀은 옵션**: 대부분의 팀은 공식룰로 충분. 진입장벽 낮춤
- **공식룰 → 커스텀 전환은 쉽게**: 공식룰 복사 후 수정 (처음부터 빈 화면 X)
- **커스텀 → 공식룰 초기화 가능**: 실수로 꼬였을 때 원클릭 복구
- **미리보기로 이해도 확보**: "이렇게 하면 몇 점?" 시뮬레이션
- **기존 경기에 소급 적용 안 함**: 변경은 다음 경기부터 적용 (과거 기록 보호)
- **자동 vs 수동 부여 분리**: 클린시트(자동), 크로바(수동) 등 유연한 설정
- **변경 이력 저장**: 언제 누가 어떤 규칙을 바꿨는지 로그 (분쟁 방지)
